# webpack系列——Code Split(代码分割)

在实际的 SPA 开发过程中，打包后我们会遇到单个文件体积过大的现象，这样的文件加载时间较长，可能导致首屏加载缓慢，用户体验不佳的问题😭

  • 那针对上面的问题，我们可以从减小文件体积、抽离公用的第三方包等方面着手，那这里我们的 Code Split 就派上用场了😎

⭐常见的 Code Split 方式

  1. entry 配置多文件入口，手动分离代码；

  2. 使用 SplitChunksPlugin 对代码块进行去重和分离；

  3. 通过模块中的内联函数动态导入；

entry 配置多文件入口

  • webpack.config.js 中，配置 entry 属性：

  • module.exports = {
  // 配置多入口文件，一个入口文件就会打包为一个 chunk
  entry: {
    index: "./index.js",
    tool: "./tool.js"
  }
    ...
}

  • 这种方式带来两个问题：

    1. 由于打包的两个文件中，都引入了一个第三方库，所以打包的时候出现了重复打包；
    2. 不能动态地将核心应用程序逻辑中的代码拆分出来；

#

⭐使用 SplitChunksPlugin 对代码块进行去重和分离；

  • 使用 webpack 的 SplitChunksPlugin 插件，能够解决重复打包模块代码的问题；

  • 在 webpack.config.js 中，配置 optimization 属性

    const {resolve} = require("path")
const HtmlWebpackPlugin = require("html-webpack-plugin")

module.exports = {
  // 配置多入口文件，一个入口文件就会打包为一个 chunk
  entry: {
    index: "./index.js",
    tool: "./tool.js"
  },
  output: {
    filename: "[name].bundle.js",
    path: resolve(__dirname,"./build")
  },
    // 使用 SplitChunksPlugin 插件
  optimization: {
      // 将重复使用的包，单独打包为一个 chunk
    splitChunks: {
      chunks: "all"
    }
  }
    ...
}

⭐通过模块中的内联函数动态导入；<

  • 对需要动态导入的模块，使用 import() 语法
import _ from 'lodash'

console.log(
  _.join(['Another', 'module', 'loaded!'], ' ')
);

// 使用 import 语法，动态导入模块，该模块将会被单独打包
import('./dynamic')
