# 什么闭包,闭包有什么用

  • 闭包是在某个作用域内定义的函数，它可以访问这个作用域内的所有变量
  • 闭包作用域链通常包括三个部分：
    • 函数本身作用域。
    • 闭包定义时的作用域。
    • 全局作用域
  • 闭包常见用途：
    • 创建特权方法用于访问控制
    • 事件处理程序及回调
    • 防止变量污染

# 为什么JS闭包会造成内存泄漏

  • 闭包延长了变量的作用域，当我们没法用JS引用变量时，垃圾回收机制仍然认为我们的变量被引用，不会被回收
  • 上述原因导致了内存被占用的越来越多

# 原型的理解

  • 定义的对象中包含一些预定义的属性
  • 函数有prototype，狭义对象有_proto_,他们都有一个constructor属性指向构造函数本身

# new运算符的执行过程（简称new的四步走）

  • 新生成一个对象
  • 连接到原型：obj.proto = Fn.prototype
  • 绑定this：apply
  • 返回新对象

# javascript有哪几种方法定义函数

  • 函数声明表达式
  • function操作符
  • Function 构造函数
  • ES6:arrow function

# this的绑定规则

  • 显式绑定
    • call/apply/bind
    • 传入null、undefined来硬绑定的时候，实际上应用的是默认绑定规则
  • 隐式绑定
    • obj.foo()
  • new绑定
    • new Function()
    • 如果构造函数中没有返回对象或者方法，this指向new出来的实例对象，反之指向返回的对象或者方法
  • 箭头函数
    • 箭头函数没有自己的 this, 它的this继承于上一层代码块的this
